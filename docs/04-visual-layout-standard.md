# استاندارد بصری

## قالب ثابت

اگر کاربر ابعاد صفحه را مشخص کند، همان ابعاد اعمال می‌شود. در غیر این صورت پیش‌فرض **A4 Portrait** با نسبت 210:297 است. قرارداد جانمایی، Grid پایه‌ی 12 ستون × 17 نوار افقی، حاشیه‌ی ثابت و ناحیه‌های ثابت عنوان، بدنه، نتیجه و پانوشت است.

Mermaid واحد چاپی دقیق را تضمین نمی‌کند؛ بنابراین این شبکه قرارداد جانمایی است و در ابزار چاپ آینده باید با ابعاد واقعی صفحه اجرا شود.

## ستون‌بندی

- سه ستون: `4+4+4` ستون پایه.
- چهار ستون: `3+3+3+3` ستون پایه.
- خطوط Grid بسیار کم‌رنگ باشند.
- جهت غالب بالا به پایین و مسیرها تا حد ممکن هم‌راستا با Grid باشند.

```text
┌──────────────────── A4 PORTRAIT ────────────────────┐
│ عنوان / شناسه / هدف                                │
├───────────┬───────────┬───────────┬────────────────┤
│           │        ناحیه‌ی نمودار روی Grid         │
├───────────┴───────────┴───────────┴────────────────┤
│ نتیجه یا Decision register                         │
├─────────────────────────────────────────────────────┤
│ راهنما، Stack و شماره صفحه                         │
└─────────────────────────────────────────────────────┘
```

## Mermaid

Block Diagram با تعداد ستون صریح ترجیح دارد. fallback، Flowchart عمودی با زیربخش‌های محدود است. گره کوتاه و مسئولیت‌محور، فناوری در خط دوم، و متن یال فقط payload باشد.

## Focus boundary

در همه‌ی P2ها عناصر زیرسیستم داخل مرز ضخیم با عنوان `Focus: <parent id> — <name>` قرار می‌گیرند. ورودی، خروجی و سرویس خارجی بیرون می‌مانند.

## تحویل

فرمت پیش‌فرض Markdown + Mermaid است و برای مشاهده نباید به HTML، PNG، PDF یا دانلود وابسته باشد، مگر با درخواست صریح کاربر.

# Interview and recovery

Works with [readiness-gate.md](readiness-gate.md).

## Cycle

1. Split input into evidence, goal, actors, data, outcomes, gaps.
2. Recover real problem, stakeholder, success measure.
3. Bound system vs environment.
4. Trace data: origin → process → decision → feedback.
5. Parts by responsibility first, technology second.
6. Test necessity; merge/remove redundancy.
7. Lock input, output, ownership, boundary per part.
8. Expand to P2 without contradiction.
9. Check incomplete flows, conflicts, open loops, hidden assumptions.
10. Record confirmed / defaults / open.

## Question policy

- Scan first; marks must be honest (doc ≠ ✅).
- One main question per turn; wait for the answer.
- Never re-ask ✅.
- Ranked answers → preferences as stated.
- Assumptions → **design defaults** shown before pages — never silent.
- Creative / living / no-destiny briefs → Standard or Deep; ask about channel, growth, residue, environment actors.

## Option reconstruction

When incomplete or conflicting: 2–4 **architecturally different** options. User ranks, blends, or rejects.

## Knowledge status

- **Confirmed:** user-backed in this run
- **Design default:** revisable; must appear in Readiness summary
- **Open:** needs decision; propose default for P3

## Node / edge

Node: purpose, input, action, output, owner, success/failure.  
Edge: direction + payload.  
If you cannot fill these from ✅ + accepted defaults → keep interviewing.
